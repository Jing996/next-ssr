# Repository Guidelines

歡迎來到 BAILUCODE AI IDE 的貢獻者指南！這份文件旨在幫助開發者快速上手本專案，並確保代碼風格、測試流程和提交規範的一致性。以下是具體的指引：

---

## Project Structure & Module Organization

本專案遵循模組化與可擴展性的設計原則，以下是目錄結構的概覽：

```
.
├── src/                # 主要源碼存放位置
│   ├── assets/         # 靜態資源（如圖片、SVG 圖標）
│   ├── components/     # 可重用的 UI 組件
│   ├── styles/         # CSS 或 SCSS 文件
│   └── utils/          # 工具函數或輔助邏輯
├── tests/              # 測試相關文件
├── .bailucode.md       # 貢獻者指南
├── package.json        # Node.js 專案依賴與腳本
└── README.md           # 專案介紹與快速入門
```

- **HTML 文件**：請將 `.html` 文件放在 `src/` 根目錄下。
- **CSS/JS 文件**：分別存放在 `styles/` 和 `utils/` 文件夾中，保持分離。
- **圖標與美化**：優先使用 FontAwesome 或 SVG，避免直接使用 Emoji。

## Build, Test, and Development Commands

以下是常用的開發指令及其用途：

### 安裝依賴
```bash
npm install
```

### 啟動開發伺服器
```bash
npm run dev
```
- 啟動本地伺服器，自動監聽檔案變更並刷新頁面。

### 建構生產版本
```bash
npm run build
```
- 生成壓縮後的靜態文件，存放在 `dist/` 目錄。

### 執行測試
```bash
npm test
```
- 使用 Jest 或其他測試框架執行單元測試與集成測試。

### 格式化代碼
```bash
npm run lint
```
- 自動檢查並修復代碼格式問題。

## Coding Style & Naming Conventions

### 縮排與格式
- 使用 **2 個空格** 作為縮排。
- 檔案結尾需保留一個空行。
- 推薦使用 Prettier 進行自動格式化。

### 命名規則
- **變數與函數**：採用駝峰命名法（camelCase），例如 `getUserData()`。
- **類別**：採用帕斯卡命名法（PascalCase），例如 `UserProfile`。
- **CSS 類名**：使用短橫線分隔（kebab-case），例如 `header-container`。

### 語言偏好
- JavaScript：ES6+ 語法。
- HTML：語義化標籤，避免過多嵌套。
- CSS：推薦使用 SCSS 提高可讀性。

## Testing Guidelines

### 測試框架
- **單元測試**：Jest。
- **端到端測試**：Cypress（如有需要）。

### 測試覆蓋率
- 至少達到 **80%** 的代碼覆蓋率。
- 關鍵邏輯需撰寫完整的測試案例。

### 測試命名規則
- 測試文件命名：`[module].test.js`。
- 測試函數命名：`should [行為描述] when [條件]`，例如：
  ```javascript
  test('should return true when user is authenticated', () => {
    // 測試邏輯
  });
  ```

## Commit & Pull Request Guidelines

### Commit Message 規範
採用 [Conventional Commits](https://www.conventionalcommits.org/) 格式，結構如下：
```
<type>(<scope>): <subject>
```

- **Type**：
  - `feat`：新增功能。
  - `fix`：修復 Bug。
  - `docs`：文檔更新。
  - `style`：樣式調整（不影響邏輯）。
  - `refactor`：重構代碼。
  - `test`：新增或修改測試。
  - `chore`：構建或工具變更。

- **範例**：
  ```
  feat(auth): add login functionality
  fix(nav): resolve dropdown alignment issue
  ```

### Pull Request 要求
1. **標題清晰**：簡潔描述更改內容。
2. **描述詳細**：列出主要變更點及原因。
3. **關聯 Issue**：使用 `Fixes #123` 或 `Closes #123` 關聯相關問題。
4. **截圖或示例**：若有 UI 變更，請附上前後對比圖。

## (Optional) Security & Configuration Tips

- **敏感信息**：切勿將 API 密鑰或密碼提交至 Git，請使用環境變數管理。
- **依賴安全性**：定期執行 `npm audit` 檢查依賴漏洞。
